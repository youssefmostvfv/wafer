كيف تعمل منصة وَفير؟ (نموذج DFD)

نظام معلومات متكامل يوضح مسارات تدفق البيانات بين الكيانات الخارجية والنظام لإدارة طلبات التمويل بكفاءة.

1

تقديم طلب التمويل

يقوم المستخدم بإنشاء حساب وإدخال بيانات طلب التمويل ودراسات الجدوى ليتم حفظها في قاعدة بيانات المشروعات (D2).

2

مراجعة الطلب

يتلقى "موظف المراجعة" الطلبات عبر النظام لدراستها فنياً واتخاذ قرار دقيق بالقبول أو الرفض وتحديث حالة المشروع.

3

استعراض واختيار المشروع

يتيح النظام للمستثمر استعراض بيانات المشروعات المعتمدة المتاحة واختيار المشروع المناسب لتوجهاته الاستثمارية.

4

تنفيذ عملية الدفع

يتم تنفيذ المعاملة المالية عبر بوابة دفع خارجية (Payment Gateway) وإصدار تأكيد الدفع للمستثمر وصاحب المشروع.

عن النظام: تحليل وتصميم منصة وَفير

النظام (System) هو مجموعة من المكونات المترابطة التي تعمل معاً كبيئة تقنية تربط بين الممولين ورواد الأعمال. يتم تحويل البيانات المدخلة مثل دراسات الجدوى واحتياجات المشروعات إلى نتائج ومعلومات مفيدة من خلال العمليات الداخلية لبرمجياتنا.

تم الاعتماد على هندسة نظم المعلومات (Information System) لتوفير تخزين منظم، واسترجاع سريع للبيانات (DBMS)، ودعم العمليات التشغيلية واتخاذ القرارات الاستراتيجية التي تفيد كافة أطراف العملية التمويلية.

معايير وتصنيفات المشروعات

كيف نصنف المشروعات وفقاً للمعايير الأكاديمية والتمويلية؟

التصنيف حسب عدد العاملين

الأكثر استخداماً؛ يشمل المشروعات متناهية الصغر (أقل من 10 عمال)، والصغيرة (من 10 لـ 50)، والمتوسطة (من 50 لـ 200 عامل).

التصنيف حسب رأس المال

يُستخدم غالباً في السياسات التمويلية، وينقسم لمشروعات ذات رأس مال محدود، وأخرى برأس مال متوسط تعتمد على قروض صغيرة أو تمويل شخصي.

التصنيف حسب النشاط

تقسم المشروعات إلى مشاريع إنتاجية (صناعية/زراعية)، مشاريع خدمية (سياحية/صحية)، ومشاريع تجارية مختصة بالبيع والتوزيع.

معيار مستوى التكنولوجيا

مشاريع تقليدية، وأخرى حديثة قائمة كلياً على الابتكار والتطوير التقني وتلبي متطلبات الاقتصاد المعرفي الحديث.

معيار المبيعات والإيرادات

مؤشر على القوة الاقتصادية والقدرة التنافسية للمشروع داخل السوق الإقليمي، حيث يشترط ألا تتجاوز إيرادات المشروع الصغير حداً معيناً (50 مليون جنيه).

أهمية تعدد المفاهيم

يساعد تصنيف المشاريع بدقة في المنصة على تحديد التحديات ووضع سياسات اقتصادية وتمويلية مناسبة لكل نوع من المشروعات.

تطوير مستدام

الأسئلة التقنية وهيكلة النظام

إجابات حول البناء التقني وتحليل قواعد البيانات لمنصة وَفير

تم استخدام تقنيات الـ Front-End مثل HTML لبناء هيكل الصفحات، و CSS لتنسيق واجهة المستخدم وجعلها متجاوبة، بالإضافة إلى JavaScript لإضافة السلوك الديناميكي والتفاعل. أما للواجهة الخلفية (Back-End) فقد تم استخدام لغة PHP لمعالجة البيانات، ونظام MySQL لإدارة قواعد البيانات وتخزينها.

تهدف عملية تصميم قاعدة البيانات إلى تحقيق التكامل بين بيانات المستخدمين والمشروعات، تقليل الازدواجية والتكرار (عبر عملية التطبيع Normalization)، ضمان دقة وسلامة البيانات الحساسة، ودعم الإدارة في اتخاذ قرارات مالية واستراتيجية سليمة.

يعتمد النظام الفيزيائي لقاعدة البيانات على تشفير البيانات الحساسة وحمايتها من الاختراق، مع تحديد صلاحيات وصول صارمة للمستخدمين (ممولين، أصحاب مشروعات، موظفي مراجعة). يتم تسجيل جميع العمليات التمويلية بدقة وحفظ سجلات السداد لضمان الشفافية والموثوقية.

تم تطوير الكود البرمجي باستخدام محرر النصوص VS Code لمرونته وقابليته للتخصيص، وتم تشغيل واختبار النظام محلياً باستخدام حزمة XAMPP التي توفر بيئة متكاملة تحتوي على خادم Apache ومحرك PHP وقاعدة بيانات MySQL.
